# PBL AI DASHBOARD - 機能説明書（プレゼンテーション用）

## 📱 アプリケーション概要

**PBL AI DASHBOARD**は、Project-Based Learning（PBL）におけるチーム活動を支援する、AI搭載のプロジェクトマネジメントダッシュボードです。学生の状態を可視化し、プロジェクトマネージャー（PM）が適切な支援を提供できるようサポートします。

**アプリURL**: https://pbl-dashboard.vercel.app/

---

## ✅ 現在実装済みの機能

### 1. **ダッシュボード機能**
- **2軸マップ可視化**
  - モチベーション × タスク量の2軸で学生の状態を可視化
  - 色分けによる危険度の直感的な表示（赤＝高リスク、黄＝注意、緑＝良好）
- **リアルタイムデータ表示**
  - 学生一覧とそれぞれの状態（モチベーション、負荷、危険度）
  - チーム別の進捗状況
  - タスクの完了状況

### 2. **学生管理機能**
- **詳細プロフィール表示**
  - MBTI性格診断結果
  - 動物タイプ診断（ライオン、コアラ、イルカ、オオカミ、カメ）
  - スキル評価（12種類のスキル：企画、実行、調整、探索、デザイン、開発、分析、ドキュメント作成、コミュニケーション、リーダーシップ、プレゼンテーション、問題解決）
  - 強み・弱み分析
  - チーム内の相性情報
- **Airtable連携**
  - 学生データをAirtableから自動取得
  - リアルタイムでのデータ同期

### 3. **AI搭載機能（ルールベース）**
以下のAIロジックが実装済みです：

#### 3.1 モチベーション推定AI
- タスク完了率、チーム相性、MBTI特性から総合的に推定
- 推定の根拠を日本語で説明

#### 3.2 負荷スコア計算AI
- タスク数、難易度、期限の緊急度から負荷を自動計算
- 負荷が高い学生を自動検知

#### 3.3 スキル自動計算AI
- タスク実績とMBTI特性からスキルを自動推定
- 12種類のスキルを5段階で評価

#### 3.4 危険度スコア計算AI
- モチベーション、負荷、期限超過タスクなどから総合的に危険度を算出
- 5段階評価（1=安全 → 5=危険）

#### 3.5 タスク再割り当て提案AI
- **AIが自動でタスク再割り当てを提案**
- 各学生の負荷、モチベーション、スキル適性を考慮
- PMが提案を**許可/拒否**で判断可能
- 許可すると**自動でWBSとAirtableを更新**
- 提案理由を詳細に説明

#### 3.6 介入推奨AI
- 危険度の高い学生に対して、PMが取るべき行動を提案
- 優先度付きで推奨アクションを表示

### 4. **プロジェクトマネージャー（PM）専用機能**
- **危険度ランキング**
  - 全学生を危険度順に表示
  - 緊急対応が必要な学生を一目で把握
- **遅延タスクアラート**
  - 期限超過タスクを自動検知
  - 優先度別に表示（緊急/重要/中程度/低）
  - 遅延の原因をAIが分析
  - 推奨アクションを提示
- **タスク再割り当て提案**
  - AIが最適な担当者を提案
  - 許可/拒否の判断機能
  - 自動でWBS・Airtable更新
- **停滞プロジェクト検知**
  - チーム全体の進捗が遅れているプロジェクトを検知
  - 完了率、遅延率、平均モチベーションを表示
- **チーム負荷バランス分析**
  - チーム内の負荷の偏りを可視化
  - 負荷バランススコアを算出
- **チーム相性マップ**
  - チームメンバー間の相性を可視化
  - MBTI特性と個人の好み（preferred_partners）を考慮
- **リーダー支援必要度**
  - チームリーダーへのサポートが必要な状況を検知
  - リーダー自身の状態とチーム全体の状況を総合評価

### 5. **WBS（Work Breakdown Structure）管理**
- **WBSファイルのアップロード**
  - CSV/JSON形式のWBSファイルをインポート
  - 複数のWBSを管理可能
- **WBS切り替え機能**
  - 複数のWBSを切り替えて表示
  - 選択したWBSに基づいてタスクと負荷を自動再計算
- **AI活用方法の自動生成**
  - 各タスクに対して、AIをどう活用すべきかを自動提案
  - タスクのカテゴリと難易度に応じて最適なAIツールを推奨

### 6. **データ管理**
- **Airtable統合**
  - 学生データをAirtableから取得
  - サンプルデータの自動投入機能
  - 画像認識で学生データを一括インポート（開発中）
- **ローカルデータフォールバック**
  - Airtableが利用できない場合、ローカルJSONファイルを使用
  - オフラインでも基本機能が動作

### 7. **初期セットアップ機能**
- **スキル自己評価**
  - 学生が12種類のスキルを自己評価
  - 自信度も併せて記録
  - 各スキルの理由を記述可能
- **チェックイン機能**
  - 日々のモチベーション、エネルギーレベル、ストレスレベルを記録
  - タスク進捗、チームコミュニケーション、個人的問題の状況を記録
  - 達成事項と課題を記録

## 🎯 技術的な特徴

### アーキテクチャ
- **フロントエンド**: Next.js 14 (App Router) + React 18 + TypeScript
- **スタイリング**: Tailwind CSS（Apple風のモダンなデザイン）
- **データベース**: Airtable（NoSQLクラウドデータベース）
- **AI**: ルールベースAI + ChatGPT API（統合済み）
- **デプロイ**: Vercel（自動CI/CD）

### AI機能の実装
- **ルールベースAI**: 7つの主要ロジック（モチベーション、負荷、スキル、危険度、タスク再割り当て、介入推奨、AI活用提案）
- **ChatGPT統合**: 統合済み（OpenAI API）
- **知識ベース**: 教育・AI関連の4冊の書籍内容を統合

### データフロー
1. Airtableから学生データを取得
2. WBSファイルからタスクデータを取得
3. AIが各種スコアを自動計算
4. ダッシュボードで可視化
5. PMが介入判断
6. Airtable・WBSを自動更新

## 📊 実証実験の進め方

### 準備
1. Airtableにデータを投入（学生情報）
2. WBSファイルをアップロード（タスク情報）
3. 各学生がスキル自己評価を実施

### 運用
1. 毎日のチェックイン（学生）
2. ダッシュボードでの状態確認（PM）
3. AIの提案に基づいた介入（PM）
4. タスク再割り当ての実施（必要に応じて）

### 評価
1. 学生のモチベーション推移
2. タスク完了率
3. プロジェクト成功率
4. AIの提案精度
5. PMの介入効果
