# 知識ベース統合ガイド

PBL AI DashboardのAI機能に、以下の4冊の書籍の知識を統合しました。

## 📚 統合された書籍

### 1. **モチベーション3.0**（ダニエル・ピンク）

**主要概念:**
- **自律性（Autonomy）**: 何を、いつ、どのように、誰と行うかについて、自分で選択する自由
- **熟達（Mastery）**: 何かを上達させたい、成長したいという欲求
- **目的（Purpose）**: 自分よりも大きな何かのために貢献したいという欲求

**活用方法:**
- 学生のモチベーションが低い場合、自律性・熟達・目的の3要素を確認
- アドバイスに内発的動機づけの観点を含める

---

### 2. **ビジネス フレームワーク 図鑑**（株式会社アンド）

**70種類のフレームワークを7カテゴリに分類:**

1. **問題発見（PROBLEM FINDING）**
   - 5W1H分析、Why分析、課題の構造化

2. **分析（ANALYTICS）**
   - SWOT分析、3C分析、PEST分析、バリューチェーン分析

3. **サービス開発（SERVICE DEVELOPMENT）**
   - カスタマージャーニー、ペルソナ設計、MVP、デザイン思考

4. **業務改善（BUSINESS IMPROVEMENT）**
   - PDCAサイクル、カイゼン、プロセス改善

5. **アイデア創出（CREATING IDEAS）**
   - ブレインストーミング、マインドマップ、SCAMPER法

6. **戦略立案（STRATEGY PLANNING）**
   - ビジョン・ミッション設定、戦略マップ、ロードマップ作成

7. **ファシリテーション（FACILITATION）**
   - 会議設計、合意形成、対話促進、チームビルディング

**活用方法:**
- 学生のスキルや状況に応じて適切なフレームワークを自動提案
- タスクのカテゴリに応じたフレームワークを推奨

---

### 3. **1on1の基本と実践がよくわかる本**（寺内健朗、島田友和）

**主要スキル:**
- **傾聴（Active Listening）**: メンバーの話を受けとめる
  - 相槌を打つ、要約して確認、感情を理解、判断を保留
- **質問（Questioning）**: 話を整理し気づきを引き出す
  - オープンクエスチョン、未来質問、リフレーミング
- **フィードバック**: 建設的なフィードバックを提供
  - SBI（状況・行動・影響）、ポジティブフィードバック

**活用方法:**
- PMやメンターへのアドバイスに1on1スキルを組み込む
- 学生との対話方法を具体的に提案

---

### 4. **学び3.0**（信岡良亮）

**主要概念:**
- **シフト**: 個人が生き残るための学び → みんなでよりよく生きるための学び
- **特徴**: 地域を旅する大学、共創型の学習、実践的な学び、社会との接続
- **原則**: 個人の成長だけでなく、社会全体の成長を目指す

**活用方法:**
- チーム全体の成長を意識したアドバイス
- 個人だけでなく、チームへの貢献を促す

---

## 🤖 AI機能での活用

### 学生アドバイス生成

**統合内容:**
- モチベーション3.0の3要素（自律性・熟達・目的）を考慮
- 状況に応じたビジネスフレームワークを自動提案
- 1on1スキル（傾聴・質問・フィードバック）を活用したアドバイス
- 学び3.0の観点（個人→チーム全体の成長）を含む

**出力例:**
```json
{
  "summary": "モチベーションが低下しています。自律性・熟達・目的の3要素を確認しましょう。",
  "detailed_advice": "1on1ミーティングで傾聴と質問スキルを活用し、内発的動機を引き出しましょう...",
  "action_items": [
    "SWOT分析で現状を整理する",
    "PMと1on1ミーティングを設定（傾聴・質問スキル活用）",
    "プロジェクトの目的を再確認する（モチベーション3.0）"
  ],
  "frameworks_to_use": [
    "問題発見フレームワーク: 5W1H分析",
    "分析フレームワーク: SWOT分析"
  ],
  "encouragement": "個人の成長だけでなく、チーム全体の成長に貢献していきましょう（学び3.0）"
}
```

### タスク分析

**統合内容:**
- タスクのカテゴリに応じた適切なフレームワークを提案
- ビジネスフレームワーク図鑑の70種類から最適なものを選択

**出力例:**
```json
{
  "optimal_ai_tools": [...],
  "recommended_frameworks": [
    {
      "name": "SWOT分析",
      "category": "分析",
      "how_to_use": "このタスクの強み・弱み・機会・脅威を整理する"
    },
    {
      "name": "カスタマージャーニー",
      "category": "サービス開発",
      "how_to_use": "ユーザーの体験を可視化して改善点を発見する"
    }
  ],
  "breakdown": {...},
  "tips": [
    "SWOT分析で現状を整理してから実装に進む",
    "カスタマージャーニーでユーザー視点を確保"
  ]
}
```

## 📊 自動提案ロジック

### 学生の状況に応じたフレームワーク提案

| 状況 | 推奨フレームワーク |
|------|------------------|
| モチベーション低 | モチベーション3.0、1on1スキル |
| 負荷高 | 業務改善（PDCA）、問題発見（5W1H） |
| 企画スキル低 | 戦略立案、アイデア創出 |
| 調整スキル低 | ファシリテーション、1on1 |
| 探索スキル低 | 分析（SWOT、3C）、サービス開発 |

## 🔧 実装ファイル

- **`frontend/lib/ai/knowledge_base.ts`**: 知識ベースの定義
- **`frontend/lib/ai/student_advisor.ts`**: 学生アドバイス生成（知識ベース統合済み）
- **`frontend/lib/ai/task_analyzer.ts`**: タスク分析（知識ベース統合済み）

---

## 💡 使用例

### APIエンドポイント

**学生アドバイス:**
```
GET /api/students/{student_id}/ai-advice
```

**タスク分析:**
```
GET /api/tasks/{task_id}/ai-analysis
```

### レスポンスに含まれる新フィールド

- `frameworks_to_use`: 推奨フレームワークのリスト
- `recommended_frameworks`: タスク分析でのフレームワーク提案（詳細付き）

3. **包括的な視点**: 個人の成長だけでなく、チーム全体の成長を意識
