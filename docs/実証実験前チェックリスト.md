# 実証実験前チェックリスト

実証実験を開始する前に、以下の項目を確認してください。

---

## 🔧 必須修正項目

### 1. エラーハンドリングの確認

- [ ] **APIエンドポイントのエラーハンドリング**
  - すべてのAPIエンドポイントで適切なエラーレスポンスを返しているか
  - 500エラーが発生した場合の適切な処理
  - タイムアウト処理の実装

- [ ] **フロントエンドのエラーハンドリング**
  - データ取得失敗時の表示
  - ネットワークエラー時の処理
  - ユーザーフレンドリーなエラーメッセージ

**確認箇所:**
- `frontend/app/api/**/*.ts` - すべてのAPIエンドポイント
- `frontend/app/**/page.tsx` - すべてのページコンポーネント

### 2. データの整合性チェック

- [ ] **Airtable接続の確認**
  - 環境変数が正しく設定されているか
  - Airtable APIキーの権限が適切か
  - Base IDが正しいか

- [ ] **データの必須フィールドチェック**
  - 学生データに必須フィールドが含まれているか
  - タスクデータに必須フィールドが含まれているか
  - チームデータに必須フィールドが含まれているか

**確認コマンド:**
```bash
node scripts/check-sample-data.js
```

### 3. セキュリティの確認

- [ ] **環境変数の保護**
  - `.env.local` が `.gitignore` に含まれているか
  - APIキーがコードに直接記述されていないか
  - 本番環境での環境変数設定方法が明確か

- [ ] **APIキーの管理**
  - Airtable APIキーの権限が最小限か
  - OpenAI APIキーの使用制限が設定されているか

### 4. パフォーマンスの確認

- [ ] **データ取得の最適化**
  - 不要なデータ取得がないか
  - キャッシュが適切に使用されているか
  - タイムアウト設定が適切か

- [ ] **ページ読み込み速度**
  - 初回読み込みが5秒以内か
  - 大量データの処理が適切か

## 🐛 既知の問題と対処法

### 1. Airtable接続エラー

**症状:**
- データが表示されない
- エラーメッセージが表示される

**対処法:**
1. `.env.local` ファイルを確認
2. APIキーとBase IDが正しいか確認
3. Airtableの権限設定を確認
4. ネットワーク接続を確認

### 2. ChatGPT API接続エラー

**症状:**
- AIアドバイスが生成されない

**対処法:**
1. OpenAI APIキーが設定されているか確認
2. 使用制限に達していないか確認
3. フォールバック機能が動作するため、アプリは継続して動作

### 3. WBSアップロードエラー

**症状:**
- ファイルがアップロードできない
- エラーメッセージが表示される

**対処法:**
1. ファイル形式（CSV/JSON）を確認
2. 必須列が含まれているか確認
3. 文字エンコーディング（UTF-8）を確認
