# スコア評価ルーブリック

## 概要
PBL AIダッシュボードでは、学生の状態を1〜5のスコアで評価します。このルーブリックは、各スコアの基準を明確化し、統一された評価を行うためのものです。

---

## 1. タスク量スコア（Load Score）

タスク量スコアは、学生が現在抱えているタスクの負荷を1〜5で評価します。

### 評価基準

| スコア | レベル | 定義 | 具体的な状態 |
|--------|--------|------|--------------|
| **1** | 非常に低い | タスクがほとんどない、または非常に軽いタスクのみ | • アクティブなタスクが0〜1件<br>• すべてのタスクが完了済み<br>• 難易度が低い（1〜2）タスクのみ<br>• 締切まで余裕がある（1週間以上） |
| **2** | 低い | タスクはあるが、余裕を持って対応可能 | • アクティブなタスクが2〜3件<br>• 難易度が中程度（2〜3）のタスク<br>• 締切まで十分な時間がある（3日以上）<br>• タスクの合計負荷が基準値の50%以下 |
| **3** | 標準 | 適切な量のタスクを抱えている | • アクティブなタスクが3〜5件<br>• 難易度が中程度（3）のタスクが中心<br>• 一部のタスクが締切に近い（1週間以内）<br>• タスクの合計負荷が基準値の50〜100% |
| **4** | 高い | タスクが多く、負荷が高い状態 | • アクティブなタスクが5〜8件<br>• 難易度が高い（4〜5）のタスクを含む<br>• 複数のタスクが締切に近い（3日以内）<br>• 期限超過のタスクがある<br>• タスクの合計負荷が基準値の100〜150% |
| **5** | 非常に高い | 過剰なタスク負荷で、対応が困難な状態 | • アクティブなタスクが8件以上<br>• 難易度が高い（4〜5）のタスクが複数<br>• 多くのタスクが締切直前または期限超過<br>• 緊急度の高いタスクが複数ある<br>• タスクの合計負荷が基準値の150%以上 |

### 計算要素

タスク量スコアは以下の要素から計算されます：

1. **基本負荷**: タスクの難易度（1-5）× 推定時間の重み
2. **緊急度マルチプライヤ**:
   - 期限超過: 2.0倍
   - 1日以内: 1.8倍
   - 3日以内: 1.5倍
   - 1週間以内: 1.2倍
3. **タスク数マルチプライヤ**: 3タスクを基準に最大1.5倍

### 推奨アクション

- **スコア1-2**: 新しいタスクを割り当てることを検討
- **スコア3**: 現状維持、定期的なモニタリング
- **スコア4**: タスクの優先順位付け、期限の調整、支援の提供を検討
- **スコア5**: 緊急の介入が必要。タスクの再分配、期限の延長、追加リソースの提供を検討

## 2. モチベーションスコア（Motivation Score）

モチベーションスコアは、学生のプロジェクトへの意欲・関心度を1〜5で評価します。

### 評価基準

| スコア | レベル | 定義 | 具体的な状態 |
|--------|--------|------|--------------|
| **1** | 非常に低い | プロジェクトへの意欲が極めて低い状態 | • タスク完了率が20%未満<br>• 進行中のタスクがない、または停滞している<br>• 自分の強みを活かせるタスクが少ない<br>• チームメンバーとの相性が悪い<br>• 活動の記録がほとんどない |
| **2** | 低い | 意欲が低く、積極的な参加が見られない | • タスク完了率が20〜40%<br>• 進行中のタスクが少ない（1〜2件）<br>• 自分の強みを活かせるタスクが一部のみ<br>• チームメンバーとの相性がやや悪い<br>• 活動の記録が少ない |
| **3** | 標準 | 適切な意欲を持ち、通常通り活動している | • タスク完了率が40〜70%<br>• 進行中のタスクが適切な数（2〜4件）<br>• 自分の強みを活かせるタスクが一部ある<br>• チームメンバーとの相性が普通<br>• 活動の記録が標準的 |
| **4** | 高い | 意欲が高く、積極的に活動している | • タスク完了率が70〜90%<br>• 進行中のタスクが複数ある（3〜5件）<br>• 自分の強みを活かせるタスクが多い<br>• チームメンバーとの相性が良い<br>• 活動の記録が多く、積極的な参加が見られる |
| **5** | 非常に高い | 極めて高い意欲で、リーダーシップを発揮している | • タスク完了率が90%以上<br>• 進行中のタスクが多く、自発的に取り組んでいる<br>• 自分の強みを活かせるタスクが非常に多い<br>• チームメンバーとの相性が非常に良い<br>• 活動の記録が多く、他のメンバーをサポートしている |

### 計算要素

モチベーションスコアは以下の要素から計算されます（重み付き平均）：

1. **タスク完了率（重み40%）**: 完了タスク数 / 総タスク数
2. **強みマッチ度（重み25%）**: 自分の強みカテゴリとタスクカテゴリの一致度
3. **チーム相性（重み20%）**: 好ましいパートナーとの協働、避けたいパートナーとの接触
4. **MBTI特性（重み15%）**: 
   - EN系（外向的・直感的）: ベーススコア 4.0
   - ES系（外向的・感覚的）: ベーススコア 3.5
   - IN系（内向的・直感的）: ベーススコア 3.0
   - IS系（内向的・感覚的）: ベーススコア 2.5

### 推奨アクション

- **スコア1-2**: 緊急の介入が必要。個別面談、タスクの見直し、チーム編成の検討
- **スコア3**: 現状維持、定期的なモニタリング、必要に応じてサポート
- **スコア4-5**: 良好な状態。リーダーシップを発揮してもらう、他のメンバーのサポートを依頼

## 3. 危険度スコア（Danger Score）

危険度スコアは、学生がプロジェクトから離脱するリスクを1〜5で評価します。

### 評価基準

| スコア | レベル | 定義 | 推奨アクション |
|--------|--------|------|----------------|
| **1** | 安全 | リスクが低く、良好な状態 | 定期的なモニタリングで十分 |
| **2** | 注意 | 軽微なリスクがある | 状況を観察し、必要に応じてサポート |
| **3** | 警告 | 中程度のリスクがある | 個別面談、タスクの見直しを検討 |
| **4** | 危険 | 高いリスクがある | 緊急の介入が必要。タスクの再分配、支援の提供 |
| **5** | 極めて危険 | 極めて高いリスクがある | 即座の介入が必要。プロジェクトマネージャーと相談 |

### 計算要素

危険度スコアは以下の要素から計算されます（重み付き平均）：

1. **モチベーションリスク（重み30%）**: モチベーションスコアが低いほど危険度が高い
2. **負荷リスク（重み25%）**: タスク量スコアが高いほど危険度が高い
3. **期限超過タスク（重み20%）**: 期限超過タスクが多いほど危険度が高い
4. **スキルギャップ（重み10%）**: 必要なスキルが不足しているほど危険度が高い
5. **活動度の低下（重み10%）**: 最近の活動が少ないほど危険度が高い
6. **コミュニケーションギャップ（重み5%）**: チーム内のコミュニケーションが減少しているほど危険度が高い

---

## 4. スコアの解釈と活用

### スコアの組み合わせによる判断

- **タスク量高（4-5）× モチベーション低（1-2）**: 最も危険な状態。即座の介入が必要
- **タスク量高（4-5）× モチベーション高（4-5）**: 負荷は高いが意欲がある。期限調整や支援で対応可能
- **タスク量低（1-2）× モチベーション低（1-2）**: タスクが少ないのに意欲も低い。タスクの見直しや動機付けが必要
- **タスク量標準（3）× モチベーション標準（3）**: 理想的な状態。現状維持

### スコアの更新タイミング

- タスクの追加・完了・更新時
- 定期的な再計算（推奨: 1日1回）
- 手動での再計算（必要に応じて）

### 注意事項

- スコアは自動計算されるため、常に最新の状態を反映します
- スコアは目安であり、個別の状況を考慮した判断も重要です
- スコアが急激に変化した場合は、背景を確認してください
