# タグ・リンク最適化

## Description
指定ファイルのタグと双方向リンクを最適化します。

## Prompt
指定ファイルのタグと双方向リンクを最適化してください。

引数:
- ファイル名（省略時は現在開いているファイル）

実行内容:

### 1. タグの最適化
1. 現在のタグを分析
2. 内容に基づいて適切なタグを提案
3. 重複・冗長なタグを指摘
4. 階層化の提案

### 2. 双方向リンクの最適化
1. 現在のリンクを分析
2. 内容から関連ノートを検索
3. 追加すべきリンクを提案
4. 不要・冗長なリンクを指摘
5. リンクの粒度を最適化

タグ階層の推奨:
- `#memory` - Memory Notes（長期記憶）
- `#input` - Input Notes（短期記憶）
- `#inbox` - Inbox Notes（一時メモ）
- `#meeting` - MTG記録
- `#project/名前` - プロジェクト別
- `#status/draft` - ステータス
- `#technology/分野` - 技術分野別
- `#ai/tools/cursor` - AIツール別（階層化）

タグのルール:
- 最大5タグまで
- 階層化タグを使用: `#technology/frontend/react`
- ファイルの場所に応じて適切なタグを提案

リンクのルール:
- **リンクすべきもの**:
  - 関連する概念（例: `[[Cursor]]` → `[[AI Coding]]`）
  - 前提知識（例: `[[RAG]]` → `[[Vector Database]]`）
  - 具体例（例: `[[Prompt Engineering]]` → `[[Prompt Examples]]`）
  - 人物・組織（例: `[[Client ABC]]` → `[[Corporate Training]]`）
- **リンク不要なもの**:
  - 一般的な単語（「今日」「明日」など）
  - 確定していない概念
  - 1回しか出てこない単語
- **粒度のルール**:
  - 1文に1-2個まで
  - 重要な概念のみリンク
  - リンク過多を避ける

出力形式:
```markdown
## タグ・リンク最適化案

### 📋 タグ最適化

現在のタグ:
#api #設計 #技術

推奨タグ:
#technology/api/design #memory

変更理由:
- 階層化でより検索しやすく
- #設計 → #technology/api/design に統合
- #memory を追加（Memoryフォルダなので）

### 🔗 双方向リンク最適化

現在のリンク:
[[API]] [[設計]] [[技術]]

追加推奨リンク:
- [[REST API設計原則]] (関連度: 95%)
  理由: 同じAPI設計の話題、前提知識として重要
  推奨挿入箇所: "設計原則"のセクション

- [[API設計パターン]] (関連度: 82%)
  理由: 具体的な設計パターンについて言及

削除推奨リンク:
- [[技術]] (理由: 一般的すぎる単語、リンク不要)

変更推奨リンク:
- [[API]] → [[REST API設計]] (理由: より具体的なノートにリンク)

### 📊 最適化サマリー
- タグ: 3個 → 2個（階層化で整理）
- リンク: 3個 → 4個（関連ノートを追加、1個削除）
- 改善点: タグの階層化、リンクの具体化
```
